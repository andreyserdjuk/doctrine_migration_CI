create_migration.php создает дамп, который, по мнению Доктрины 
нужно сейчас накатить на существующую базу. После создания дампа
желательно глянуть на него, так как в нем могуть быть ненужные
ALTERы, которые вызовут ошибку при накатывании изменений.

apply_migration.php применяет в прямом хронологическом порядке дампы, 
созданные create_migration.php начиная с текущего дампа(не включая его) - 
это значение db_version в таблице local_office.


изначально запись db_version создается в момент первого запуска 
apply_migration.php, и устанавливает значение единственного (т.е.
первого) дампа - естественно, в первый раз этот дам выполнится.
